# Pattern: Mandatory Observability Fields

Pattern ID: CL-PAT-002
Status: Active
Confidence: provisional

Context:
Cross-layer interactions require tracing, debugging, and audit capabilities.

Problem:
Requests without consistent observability fields cannot be correlated across services or traced through logs.

Solution:
All cross-layer payloads MUST include observability fields: `request_id`, `actor_id`, and `timestamp`. These fields enable tracing, auditing, and correlation.

Rules Enforced:
- CL-R-004

Enforceable Semantics:
- Every request payload (query or command) MUST include:
  - `request_id` (string, UUID v4): Unique identifier for the request, generated by the client.
  - `actor_id` (string): Authenticated user or system identifier.
  - `timestamp` (ISO 8601 string): Client-side timestamp when the request was initiated.
- Backend MUST log these fields at entry, exit, and error points.
- Response payloads MAY echo `request_id` for client-side correlation.
- Fields MUST appear at the top level of the payload or in a consistent `meta` envelope.

Consequences:
- Requests can be traced end-to-end across UI and backend.
- Audit logs are complete and correlatable.
- Debugging is faster with stable identifiers.
